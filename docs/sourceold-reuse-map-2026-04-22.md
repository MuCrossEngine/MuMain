# sourceOld reuse map (2026-04-22)

## Objective
Use `Main/sourceOld` selectively to accelerate Android portability **without** regressing the current Android-compatible flow.

## Inventory snapshot
- Common files (`source` ‚ą© `sourceOld`): **1059**
- Identical content: **929**
- Different content: **130**
- Android-only in `source`: **42** (all under `Main/source/Platform`)

## Key finding
Directly copying files from `sourceOld` is high-risk in this branch.

Observed in inspected "LOW" candidates:
- `sourceOld` still has Windows-style include separators (`\\`) in several files, while `source` is normalized (`/`).
- Some signatures in `source` are stricter/safer (ex: `const` qualifiers) and would be degraded by raw copy from `sourceOld`.

Backslash-include pattern count:
- `sourceOld`: **19 files**
- `source`: **0 files**

## Recommended migration strategy
1. Keep `Main/source` as baseline (already Android-adapted + validated).
2. Use `sourceOld` as reference for **manual merge**, not raw overwrite.
3. Prioritize only files that are:
   - outside `Platform/`
   - outside login/network core (`WSclient`, `ZzzScene`, `UIMng`, `LoginMainWin`, `ServerSelWin`, `NewUISystem`, reconnect path)
   - diff-small and with no include/path regression
4. Validate each batch with Android `assembleDebug` + smoke flow (server list -> click -> connect -> LoginWin).
